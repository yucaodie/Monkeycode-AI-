# 知识助手 - 技术设计文档

## 1. 系统架构

### 1.1 架构概述

采用 3 层精简架构，避免过度设计：

```
┌──────────────────────────────────────────┐
│          应用层 (Next.js)                 │
│  前端页面 + API Routes + Server Actions  │
│  存储逻辑 / 搜索逻辑 / AI编排逻辑        │
└──────────────┬───────────────────────────┘
               ↓
┌──────────────┴───────────────────────────┐
│              数据层                        │
│  PostgreSQL (pgvector + 全文检索)         │
│  本地文件系统 (图片/文档存储)              │
└──────────────┬───────────────────────────┘
               ↓
┌──────────────┴───────────────────────────┐
│            AI 服务层 (内网 vLLM)           │
│  文本模型 | 多模态模型 | 嵌入向量模型      │
└──────────────────────────────────────────┘
```

### 1.2 设计原则

1. **简洁优先**：3 层架构，不引入多余中间层
2. **接口抽象**：搜索接口定义抽象层，便于未来迁移 ElasticSearch
3. **配置驱动**：AI 模型端点通过配置文件管理，支持热切换
4. **渐进增强**：MVP 聚焦核心功能，预留扩展点

## 3. 数据模型设计

### 3.1 核心表结构

```sql
-- 知识片段表
CREATE TABLE knowledge_items (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    title VARCHAR(500) NOT NULL,
    content TEXT,
    content_type VARCHAR(20) NOT NULL DEFAULT 'markdown',
    item_type VARCHAR(20) NOT NULL DEFAULT 'text',
    summary TEXT,
    embedding vector(1536),
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 附件表（图片、文档等）
CREATE TABLE attachments (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    item_id UUID NOT NULL REFERENCES knowledge_items(id) ON DELETE CASCADE,
    file_name VARCHAR(500) NOT NULL,
    file_path VARCHAR(1000) NOT NULL,
    file_type VARCHAR(50) NOT NULL,
    file_size BIGINT NOT NULL DEFAULT 0,
    extracted_text TEXT,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 标签表
CREATE TABLE tags (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(100) NOT NULL UNIQUE,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 知识片段-标签关联表
CREATE TABLE item_tags (
    item_id UUID NOT NULL REFERENCES knowledge_items(id) ON DELETE CASCADE,
    tag_id UUID NOT NULL REFERENCES tags(id) ON DELETE CASCADE,
    source VARCHAR(20) NOT NULL DEFAULT 'manual',
    PRIMARY KEY (item_id, tag_id)
);

-- 双向链接表
CREATE TABLE item_links (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    source_item_id UUID NOT NULL REFERENCES knowledge_items(id) ON DELETE CASCADE,
    target_item_id UUID NOT NULL REFERENCES knowledge_items(id) ON DELETE CASCADE,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (source_item_id, target_item_id)
);

-- AI 对话会话表
CREATE TABLE chat_sessions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    title VARCHAR(500),
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- AI 对话消息表
CREATE TABLE chat_messages (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id UUID NOT NULL REFERENCES chat_sessions(id) ON DELETE CASCADE,
    role VARCHAR(20) NOT NULL,
    content TEXT NOT NULL,
    referenced_item_ids UUID[],
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

### 3.2 索引设计

```sql
-- 全文检索索引（中英文）
CREATE INDEX idx_knowledge_items_fts ON knowledge_items
    USING gin(to_tsvector('simple', coalesce(title, '') || ' ' || coalesce(content, '') || ' ' || coalesce(summary, '')));

-- 向量索引（IVFFlat，适合中小规模）
CREATE INDEX idx_knowledge_items_embedding ON knowledge_items
    USING ivfflat (embedding vector_cosine_ops) WITH (lists = 100);

-- 时间线索引
CREATE INDEX idx_knowledge_items_created_at ON knowledge_items (created_at DESC);

-- 标签查询索引
CREATE INDEX idx_item_tags_tag_id ON item_tags (tag_id);
CREATE INDEX idx_item_tags_item_id ON item_tags (item_id);

-- 双向链接索引
CREATE INDEX idx_item_links_source ON item_links (source_item_id);
CREATE INDEX idx_item_links_target ON item_links (target_item_id);
```

### 7.2 AI 整理输出流程

```
用户描述需求格式
       ↓
生成需求描述的嵌入向量
       ↓
pgvector 语义检索相关知识片段
       ↓
组装 Prompt（系统指令 + 相关知识 + 输出格式要求）
       ↓
调用 vLLM 文本模型生成
       ↓
返回格式化结果
```

### 7.3 图片理解流程

```
用户上传图片
       ↓
保存到本地文件系统
       ↓
调用 vLLM 多模态模型理解图片内容
       ↓
提取的文字/描述存入 knowledge_items
       ↓
生成嵌入向量用于语义搜索
```

## 8. 部署方案

### 8.1 Docker Compose

```yaml
version: '3.8'

services:
  app:
    build: .
    ports:
      - "3000:3000"
    environment:
      - DATABASE_URL=postgresql://postgres:postgres@db:5432/knowledge
      - AI_CONFIG_PATH=/app/config/ai.yaml
      - UPLOAD_DIR=/app/data/uploads
    volumes:
      - uploads:/app/data/uploads
      - ./config:/app/config
    depends_on:
      db:
        condition: service_healthy

  db:
    image: pgvector/pgvector:pg15
    environment:
      - POSTGRES_DB=knowledge
      - POSTGRES_USER=postgres
      - POSTGRES_PASSWORD=postgres
    volumes:
      - pgdata:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 5s
      timeout: 5s
      retries: 5

volumes:
  pgdata:
  uploads:
```

### 8.2 环境要求

- Docker 20.10+
- Docker Compose V2
- 内网 vLLM 服务已就绪
- 服务器最低配置：2C4G（数据库 + 应用）

## 9. 扩展性设计

### 9.1 搜索迁移路径

当前使用 PostgreSQL + pgvector，未来迁移 ElasticSearch 的路径：

1. 实现 `EsSearchProvider` 类
2. 通过配置切换 `SearchProvider` 实现
3. 数据双写过渡期（PostgreSQL + ES 同时写入）
4. 验证 ES 搜索结果准确性
5. 切换读取到 ES
6. 移除 pgvector 依赖

### 9.2 功能扩展预留

| 扩展点 | 当前实现 | 未来扩展 |
